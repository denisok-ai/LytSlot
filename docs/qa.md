# Вопросы по архитектуре и требованиям (Q&A)

Вопросы, возникшие при создании архитектуры LytSlot Pro. После ответов заказчика/команды ответы можно перенести в Project.md или Diary.md.

---

## 1. Инфраструктура и домен

1. **Домен и бренд**: Какой домен планируется для веб-кабинета и API (например, app.lytslot.ru)? Нужна ли отдельная зона для админки (admin.lytslot.ru)?

2. **Хостинг**: Окончательный выбор — Render, AWS, DigitalOcean или гибрид? Есть ли ограничения по региону данных (только РФ, EU, global)?

3. **Kubernetes vs Docker Swarm**: В описании указаны оба варианта. Какой приоритет для MVP и для масштабирования?

---

## 2. Мульти-тенантность и границы

4. **Граница tenant**: Tenant — это один физический владелец (физлицо/ИП/юрлицо) с несколькими каналами, или возможны «организации» с несколькими пользователями и ролями (owner, editor)?

5. **Лимиты Freemium**: Точные лимиты бесплатного тарифа (1 канал — подтверждено). Лимит заказов/месяц или только количество каналов?

---

## 3. Платежи и юридические требования

6. **Юрисдикция**: ЮKassa — только для резидентов РФ? Нужна ли поддержка самозанятых (иные реквизиты/чеки)?

7. **ККТУ/ERID**: Требуется ли полная поддержка с первого релиза или допустимо ввести в течение N месяцев после MVP? Кто предоставляет юрконсультацию по маркировке рекламы?

8. **Payouts**: Вывод средств владельцам каналов — только на счёт в РФ (ЮKassa) или также Stripe Connect для международных получателей?

---

## 4. Бот и Telegram

9. **Права бота**: Бот должен быть добавлен во все каналы клиентов как администратор с правом публикации. Подтверждена ли политика «один бот на всю платформу» с точки зрения лимитов Telegram API (например, количество каналов на одного бота)?

10. **Верификация канала**: Как подтверждается владение каналом — только добавление бота как админа или дополнительно верификация через Telegram (например, пост с кодом)?

---

## 5. Модерация и контент

11. **Модерация до публикации**: Обязательна ли пре-модерация каждого креатива перед публикацией, или достаточно пост-модерации и блокировки по жалобам?

12. **AI-модерация**: Языки контента (RU/EN/другие)? Нужна ли блокировка по категориям (например, 18+, запрещённые темы) с первого дня или это фаза 2?

13. **Хранение медиа**: Где хранить загруженные креативы — только CDN (Cloudinary) или также дублирование в своём S3-совместимом хранилище для аудита/юридических запросов?

---

## 6. Аналитика и метрики

14. **Источник views**: Просмотры постов — только из Telegram (если API даёт) или расчёт по другим метрикам (например, переходы по ссылкам, данные от клиента)?

15. **PostHog**: Self-hosted или облачный PostHog? Нужна ли анонимизация для GDPR/законодательства РФ?

---

## 7. Безопасность и доступ

16. **Админ-доступ**: Один суперадмин или несколько ролей (суперадмин, модератор, поддержка)? Нужна ли 2FA для админов?

17. **API-ключи**: Срок жизни ключей, ограничения по scope (только чтение, создание заказов и т.д.) и лимиты запросов по ключу.

---

## 8. Разработка и репозиторий

18. **Монорепо vs мультирепо**: Один репозиторий (backend, bot, frontend, infra в папках) или отдельные репозитории с общими библиотеками (например, общие модели/контракты)?

19. **Приоритет рынка**: MVP в первую очередь для РФ (ЮKassa, ККТУ/ERID, русский интерфейс) или параллельно international (Stripe, EN)?

20. **Команда и роли**: Кто принимает решения по продукту (приоритет фич, ответы на qa.md)? Есть ли выделенный DevOps/инфраструктура?
